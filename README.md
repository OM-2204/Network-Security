## рҹ“Ң Overview

**NetGuard AI** is a production-grade, end-to-end **MLOps system** for detecting network intrusions and classifying malicious traffic. Built on the **NSL-KDD dataset**, it implements a complete **ETL pipeline**, automated ML training with **Random Forest**, and **CI/CD deployment to AWS EC2** via Docker and GitHub Actions.

The system achieves **99.14% F1-score** across experiment runs tracked on **MLflow + DagsHub**, and ships a professional **cyberpunk-themed web dashboard** for real-time threat prediction.

> **TL;DR:** CSV network traffic вҶ’ ETL вҶ’ Train вҶ’ Predict вҶ’ Deploy to AWS. Every stage automated.

## вҡҷпёҸ Pipeline Deep Dive

### 1пёҸвғЈ Data Ingestion

Pulls raw data from **MongoDB Atlas**, exports to a feature store, drops irrelevant columns, and splits into train/test CSVs вҖ” all timestamped.

```
Config: Ingestion Dir В· Feature Store Path В· Train/Test Path В· Split Ratio В· Collection Name

MongoDB Atlas
     в”Ӯ
     в–ј
Initiate Data Ingestion
     в”ңв”Җв”Җв–ә Export to Feature Store  вҶ’  raw.csv  (timestamped)
     в”ңв”Җв”Җв–ә Drop Columns             (schema file)
     в””в”Җв”Җв–ә Split Data               вҶ’  train.csv + test.csv  (ingested/)
```

**Output Artifact:** `raw.csv` В· `train.csv` В· `test.csv`

### 2пёҸвғЈ Data Validation

Three checks ensure data quality before training:

```
в‘  Same Schema       вҶ’  Validate same number of features (10 features expected)
в‘Ў Data Drift        вҶ’  Compare train vs test distributions вҶ’ drift report
в‘ў Column Validation вҶ’  Validate column count + numerical columns exist

Data Ingestion Artifact
     в”Ӯ
     в–ј
Read Data (train.csv + test.csv)
     в”ңв”Җв”Җв–ә Validate Number of Columns  в”Җв”Җв–ә  Train Status / Test Status
     в”ңв”Җв”Җв–ә Numerical Columns Exist     в”Җв”Җв–ә  Train Status / Test Status
     в””в”Җв”Җв–ә Detect Dataset Drift        в”Җв”Җв–ә  Drift Status
               в”ңв”Җв”Җ True  в”Җв”Җв–ә Validation Status в”Җв”Җв–ә Artifact вң…
               в””в”Җв”Җ False в”Җв”Җв–ә Validation Error  вқҢ
```

**Output Artifact:** validation status В· valid/invalid train & test paths В· drift report

### 3пёҸвғЈ Data Transformation

Handles missing values, scales features, and balances class imbalance using **SMOTE-Tomek**.

```
Data Validation Artifact
     в”Ӯ
     в–ј
Read Data  вҶ’  Drop Target Column  вҶ’  Target Value Mapping
     в”Ӯ
     в”ңв”Җв”Җв–ә Handle Missing Values
     в”Ӯ         в”ңв”Җв”Җ KNN Imputer    (fills NaN using K-nearest neighbors)
     в”Ӯ         в””в”Җв”Җ Simple Imputer (fallback)
     в”Ӯ
     в”ңв”Җв”Җв–ә Robust Scaler           (scale features, outlier-resistant)
     в”Ӯ
     в”ңв”Җв”Җв–ә SMOTE-Tomek             (balance imbalanced classes)
     в”Ӯ         в”ңв”Җв”Җ fit_transform  вҶ’  Train data
     в”Ӯ         в””в”Җв”Җ transform      вҶ’  Test data
     в”Ӯ
     в””в”Җв”Җв–ә Save Preprocessor Object  вҶ’  preprocessing.pkl
               Output: train.npy + test.npy
```

**Output Artifact:** `train.npy` В· `test.npy` В· `preprocessing.pkl`


## рҹҗі CI/CD вҖ” AWS EC2 Deployment


## рҹҡҖ Getting Started (Local)

### 1. Clone & Setup

```bash
git clone https://github.com/OM-2204/Network-Security.git
cd Network-Security

# Create virtual environment
python -m venv venv
source venv/bin/activate        # Mac/Linux
venv\Scripts\activate           # Windows

```

### 2. Configure Environment

```bash
# Create .env file
echo "MONGO_DB_URL=mongodb+srv://<user>:<pass>@cluster.mongodb.net/" > .env
```

### 3. Run

```bash
python app.py
# OR
uvicorn app:app --host 0.0.0.0 --port 8000 --reload
```
